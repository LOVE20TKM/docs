# Extension Examples

## LP extension

- Repo:
  `extension-lp`
- Main files:
  `extension-lp/src/ExtensionLp.sol`
  `extension-lp/src/ExtensionLpFactory.sol`
- Purpose:
  Let LP token holders join an action and split rewards with governance-ratio limits.
- Key parameters:
  - join LP token address
  - governance ratio multiplier
  - minimum governance ratio
- Key interface reads:
  `deduction`, `totalDeduction`, `govRatio`
- Source-of-truth note:
  Use `extension-lp` source files before frontend config, periphery helpers, or cast examples when the question is about real LP extension behavior.

## Chain-group action extension

- Repo:
  `extension-group`
- Design doc:
  `extension-group/docs/链群行动扩展协议.md`
- Main files:
  `ExtensionGroupAction.sol`, `ExtensionGroupActionFactory.sol`, `GroupManager.sol`, `GroupJoin.sol`, `GroupVerify.sol`, `GroupNotice.sol`
- Purpose:
  Let participants join actions through chain groups that are activated by group NFT owners.
- Special mechanics:
  - group activation stake
  - distrust voting against group service providers
  - group-level verification and reward accounting
  - optional trial participation mode
- Source-of-truth note:
  Use `extension-group` source files for extension behavior, and switch to `group/src/LOVE20Group.sol` only when NFT ownership or group identity rules matter.

## Chain-group service extension

- Design doc:
  `extension-group/docs/链群服务扩展协议.md`
- Main files:
  `ExtensionGroupService.sol`, `ExtensionGroupServiceFactory.sol`, `GroupRecipients.sol`
- Purpose:
  Reward chain-group service providers based on the action rewards generated by the groups they serve.
- Special mechanics:
  - governance-ratio cap on service reward
  - per-group recipient splits
  - overflow burn path
- Source-of-truth note:
  Treat `ExtensionGroupService*` and `GroupRecipients.sol` as the primary behavior source for service reward routing.

## Dependency on chain-group NFT

- Repo:
  `group`
- Main file:
  `group/src/LOVE20Group.sol`
- Role:
  Own and transfer the NFT identity used by chain-group extensions.
- Boundary note:
  `group` is not the extension implementation repo; it is the NFT protocol that chain-group extensions depend on.
